# Финальный проект курса «Инженер машинного обучения» (Karpov.Courses)

Обзор
- Этот репозиторий содержит код финального проекта курса «Инженер машинного обучения» платформы Karpov.Courses.
- Проект разбит на три части (папки `finalproject1`, `finalproject2`, `finalproject3`), каждая из которых решает определённую подзадачу в пайплайне рекомендательной системы для ленты социальной сети.

Структура репозитория
- [finalproject1](finalproject1) — сервис на базе FastAPI: HTTP‑эндпойнты для работы с базой и возвращения топ N постов по числу лайков (сервисная часть).
- [finalproject2](finalproject2) — подготовка признаков и обучение классических моделей (feature engineering, обучение CatBoost и т.п.).
- [finalproject3](finalproject3) — улучшение модели с применением приёмов из блока Deep Learning: вычисление эмбеддингов с помощью Hugging Face (DistilBERT), в папке содержится ноутбук и обученная модель.

Краткое описание частей
- Часть 1 (сервис): реализует FastAPI‑приложение, которое обслуживает запросы к базе данных и возвращает рекомендации (топ N постов) — основная задача при развёртывании сервиса.
- Часть 2 (модели): содержит код для построения фичей из сырых таблиц, обучение модели ML (пример — CatBoost), оценку качества и сохранение модели и фич в базу. Также изменён метод рекомендаций, использующий обученную модель.
- Часть 3 (DL улучшения): демонстрирует, как улучшить модель с помощью эмбеддингов: вычислить векторные представления текстов через DistilBERT, снизить размерность (PCA), кластеризовать (KMeans) и использовать расстояния до центров как дополнительные признаки. В этой части — только ноутбук и модель; код сервисов в проекте не меняется.

Данные и предпосылки
- В качестве исходных данных используются подготовленные командой курса таблицы (Postgres). Ноутбуки читают эти таблицы через `read_sql`.
- Файлы внутри папок упрощены и имеют линейную структуру из соображений автоматизированной проверки на платформе — это не окончательная структура продакшен‑проекта.

Безопасность и замечания
- В репозитории для удобства разработки присутствуют настройки/пароли/строки подключения в открытом виде (так как все студенты курса их знают). На продакшене так делать нельзя — секреты нужно выносить в переменные окружения или менеджер секретов.
- При работе с `to_sql` обращайте внимание на состояние транзакции: если до записи произошло исключение, транзакцию нужно откатить или пересоздать соединение (пример показан в ноутбуках).

Как запустить локально
1. Создайте виртуальное окружение и установите зависимости:

```powershell
python -m venv .venv
& .venv\Scripts\Activate.ps1
pip install -r finalproject3/requirements.txt
```

2. Откройте соответствующий ноутбук (`finalproject2/2.ipynb` или `finalproject3/3.ipynb`) и выполняйте ячейки последовательно.
3. Для запуска сервиса откройте [finalproject1](finalproject1) и запустите FastAPI‑приложение (см. инструкции внутри папки).

Файлы и артефакты
- Ноутбуки: `finalproject2/2.ipynb`, `finalproject3/3.ipynb`.
- Сервис: код в `finalproject1` (FastAPI).
- Зависимости: `requirements.txt` в каждой папке.
